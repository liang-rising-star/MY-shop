# 商品详情页媒体缩略图加载优化

## 问题
商品详情页打开后，底部预览缩略图分3批逐个加载：图片瞬间显示 → 视频缩略图逐个请求 → 视频预加载。根本原因是每个视频缩略图都单独发一次HTTP请求到后端用OpenCV实时截帧。

## 方案：三管齐下

### 1. 后端：上传时预生成并存储缩略图
**文件**: `app/models.py`
- Product 模型新增 `video_thumbnails = Column(Text, default="")` 字段，存储与 `video_url` 一一对应的缩略图URL（逗号分隔）

**文件**: `app/routers/products.py` 上传端点（约295-320行）
- 上传视频时已经用 cv2 生成了缩略图，返回了 `thumbnail` 字段
- 无需改动（已生成）

**文件**: `static/admin.html` saveProduct 函数
- 保存商品时，从 mediaList 中提取每个视频的缩略图URL，存入 `video_thumbnails` 字段

**文件**: `static/admin.html` 上传处理
- 上传视频后，除了推入 `d.url` 到 mediaList，同时记录该视频对应的缩略图URL
- 方案：上传时把 `{url: xxx, thumbnail: yyy}` 存到一个临时映射对象中，保存商品时写入 `video_thumbnails`

### 2. 后端：批量缩略图接口
**文件**: `app/routers/products.py`
- 新增接口 `GET /api/admin/products/{pid}/thumbnails?video_urls=url1,url2,...`
- 一次性接收所有视频URL，批量生成缩略图，返回 `{thumbnails: {url1: thumb1, url2: thumb2, ...}}`
- 已有缩略图的直接返回缓存路径，不重复生成

### 3. 前端：优先读缓存 + 并行请求兜底
**文件**: `static/js/app.js` setupDetailMedia 函数（170-190行）
- 优先从 `product.video_thumbnails` 字段读取已存储的缩略图，立即填充 `videoThumbnails` ref
- 如果某个视频没有预存缩略图，才用 `Promise.all` 并行请求批量接口兜底
- 这样大部分情况下缩略图瞬间全部显示，只有新上传未保存的才需要请求

**文件**: `static/js/app.js` 预加载函数（333-347行）
- preloadMedia 保持不变（视频预加载是正常行为）

### 修改文件清单
| 文件 | 改动 |
|------|------|
| `app/models.py` | 新增 `video_thumbnails` 字段 |
| `app/routers/products.py` | 新增批量缩略图接口 |
| `static/admin.html` | 上传时记录缩略图映射 + 保存时写入 video_thumbnails |
| `static/js/app.js` | setupDetailMedia 优先读缓存 + Promise.all 兜底 |

### 验证方式
1. 上传一个带多个视频的商品 → 保存 → 再打开编辑，确认缩略图URL已存储到 video_thumbnails 字段
2. 打开前台商品详情页 → 确认所有缩略图瞬间加载，无逐个闪烁
3. 新上传视频（未保存的商品）→ 确认 Promise.all 兜底正常工作
4. 缩略图点击/自动轮播功能正常
